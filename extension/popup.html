📘 Toolbar Help Guide

What this does

  • Floating toolbar (front-end): use while viewing parish sites to capture bulletin actions.
  • Operator Console (backend): use for parish directory updates, URL overrides, and mega PDF control.

Train a new parish (step-by-step)

  1. Open the parish bulletin website.
  2. Use the floating toolbar to record click steps and the final bulletin URL.
  3. Push the recipe to GitHub from the toolbar.
  4. Open Operator Console → Parish Directory to verify status and run an instant rebuild.

Update a changed parish URL

  1. Open the new bulletin page/PDF in the active tab.
  2. Operator Console → Parish Directory → click 📌 for that parish.
  3. This overwrites the parish override URL (latest URL only).
  4. Use 🧹 later to clear the override.

Stale bulletin warning

  • If stale warning appears, click Fix to open the parish page, then re-save the correct URL/recipe.
  • After the next harvest run, the warning clears when fresh data is found.

Parish Directory

  • Browse by diocese accordion and search parish names.
  • Use parish 📁 details to see current URL, recipe changes, and last update/mega timestamps.

Mega PDF

  • The mega PDF is the merged weekly diocesan bulletin pack.
  • It updates after successful harvest workflow runs and is published to GitHub Pages.

Pattern learning (automatic)

  • When you push a recipe, the extension saves the site layout to parishes/site_patterns.json on GitHub.
  • Next time you open a similar parish site, the toolbar shows a Pattern hint with steps that worked elsewhere.
  • No API keys needed — this is rule-based, not chat AI.
